# Handoff

## Current State

High-level EPUB research is complete. The notes intentionally paraphrase broad ideas and avoid copied text, proprietary card details, exact exercise wording, and source assets. No production code was changed.

## Key Takeaways for Product Architecture

- Build around explicit responsibility ownership, not generic shared to-do lists.
- Represent the lifecycle of work so planning and follow-through are visible.
- Use check-ins as a recurring review mechanism for changing capacity and standards.
- Keep fairness language calm and non-scorekeeping.
- Let users define household-specific standards and categories.
- Treat personal time and emotional/social coordination as legitimate topics without using therapy framing.

## Handoff to IP/Safety Agents

- Do not approve a literal card deck, source-branded terms, source examples, or source exercise structures without explicit legal clearance.
- Confirm crisis, coercion, and therapy-adjacent boundaries before relationship-support copy is finalized.

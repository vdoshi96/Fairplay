# Gaps

- Product architect should decide whether v1 includes starter templates at all. If yes, templates should be original, generic, and IP-reviewed.
- Product architect should define how the app distinguishes accountable ownership from collaborative participation.
- Product architect should decide how much emotional/social coordination belongs in v1 versus later research.
- Safety reviewer should define language for tense household conversations, including when to avoid using the app as a confrontation tool.
- IP reviewer should decide whether any source-adjacent terms are acceptable in internal docs only, and which must be avoided entirely.
- Data architect should decide whether lifecycle phases are first-class fields, checklist sections, or freeform notes.
- Product architect should decide how check-in history is summarized without creating a blame ledger.
- Future research should compare non-source household-management patterns so v1 is not overly dependent on one private reference.
